L-PRISM

Sobre o L-PRISM

L-PRISM é uma linguagem de domínio que cria cadeias de serviços multimídia baseadas em sensores virtuais multimídia. O L-PRISM permite a implantação e comunicação de aplicações multimídia baseadas em tecnologias de virtualização, e por sua vez permite a utilização de aplicações multimídia desenvolvidas por terceiros, sem a necessidade de amplo conhecimento das tecnologias ou ferramentas utilizadas para desenvolver essas aplicações.

No futuro, espera-se que o L-PRISM sirva para que novas arquiteturas utilizem esta linguagem como referência e; Por sua vez, essas arquiteturas suportam aplicativos virtualizados por terceiros que utilizam essa linguagem como referência.

Resumo do L-PRISM

L-PRISM é uma linguagem composta por diferentes tipos de dados, capacidades, artefatos, relacionamentos e por fim os nós, que fazem parte da cadeia de serviços multimídia. Nesta secção apresentamos os principais elementos de uma cadeia de serviços multimédia, caso pretenda rever as especificações detalhadas 'L-PRISM - Detalhes'.

Para criar uma cadeias de serviços multimídia com L-PRISM, você deve criar um arquivo YAML(.yml) e colocar o seguinte como variáveis ​​de cabeçalho.

---
chainModel:
  chainName: Example 1 chain sample #Name of chain
  chainDescription: Description of my chain #Description of chain

'devices' - Virtual Devices

O 'device' são os dispositivos físicos virtualizados, uma das principais características dos dispositivos virtualizados (device) é que eles podem fazer parte de diferentes cadeias de serviços multimídia, é por isso que, para consumir o fluxo multimídia que eles produzem, os sensores virtuais multimídia (VMS) precisam se inscrever em dispositivos virtualizados para que eles repliquem seu fluxo e o enviem para um novo assinante.

'deviceName'

DeviceName é o nome do dispositivo virtualizado dentro da cadeia de serviços multimídia, este nome não deve ser repetido na cadeia de serviços multimídia que está sendo criada, por sua vez, este nome será utilizado para comunicação do dispositivo virtualizado com os sensores virtuais multimídia (VMS).

  • type: string
  • required: true

'deviceId'

'deviceId' é o identificador do 'tópico' onde o dispositivo físico virtualizado está trabalhando, este identificador é usado para que os elementos da cadeia de serviços multimídia (VMSs) possam se inscrever no fluxo multimídia oferecido pelos dispositivos físicos virtualizados, isto é feito através deste identificador.

Para que um sensor virtual multimídia possa se inscrever, ele deve possuir o identificador do dispositivo virtualizado (device). Na arquitetura alfa, esse identificador é o 'id' do registro no banco de dados.

  • type: lPrism.artifacts.vms.swImage
  • required: true

Exemplo de implementação (YAML)

  devices:
  -
      deviceName: example1 video ball #Name of device in chain
      deviceId: 638e70b10a1fbd0026fccaf4 #id of device (subcribe topic)

'vmss' - Virtual Multimedia Sensors (VMS)

'vmsName'

Nome do VMS dentro da cadeia de serviços multimídia, este nome serve como guia para as conexões que existirão entre VMS-VMS ou dispositivo-VMS.

  • type: string
  • required: true

'vmsType'

vmsType é uma variável que identifica qual VMS será utilizado, esta variável está diretamente relacionada com as imagens do Docker (arquitetura alfa) disponíveis para implantação.

  • type: lPrism.artifacts.vms.swImage
  • required: true

'nodeIp'

nodeIp é uma variável que define em qual endereço IP o VMS será implantado. Em um ambiente distribuído, diferentes VMS podem ser criados e estes podem ser distribuídos em diferentes nós dentro da rede.

  • type: lPrism.datatype.vms.AddressData
  • required: true

'startupParameters'

startupParameters é uma variável que permite colocar variáveis ​​adicionais que o VMS permite, isso vai depender da configuração do desenvolvedor que criou originalmente o VMS.

  • type: lPrism.datatype.vms.configurableProperties
  • required: false

'virtualMemory'

virtualMemory é a quantidade de memória RAM atribuída ao VMS. O funcionamento correto dos aplicativos multimídia virtualizados depende desta variável. Se esta variável não for definida com um valor, a maioria das ferramentas de virtualização atribui uma quantidade de memória ao VMS por padrão.

  • type: lPrism.datatype.vms.VirtualMemory
  • required: false

'virtualCPU'

virtualCPU é o número de CPUs atribuído ao VMS, o correto funcionamento dos aplicativos multimídia virtualizados depende desta variável, se você não definir um valor para esta variável, a maioria das ferramentas de virtualização atribui uma quantidade limitada de CPUs ao VMS por padrão.

  • type: lPrism.datatype.vms.VirtualCpu
  • required: false

Exemplo de implementação (YAML)

  vmss:
  -
      vmsName: vms1 example 1 #Name of VMS in chain
      vmsType: 638e70b10a1fbd0026fccae4 #image docker_id
      nodeIp: 192.168.0.117 #IP node deploy VMS 
      startupParameters:  #optional parameter
      virtualMemory:
        size: 1024 #optional (default docker configure))
      virtualCPU:
        numCpu: 1 #optional (default docker configure)

'virtualLinks' - Virtual Links

VirtualLinks são as conexões existentes entre os diferentes dispositivos dentro de uma cadeia de serviços multimídia, podendo ser desde um dispositivo-VMS, VMS-VMS e VMS até um destino final. A características de um VirtualLinks são as seguintes.

'origin'

Agrupa as características de onde a conexão vai começar.

  • 'originName'

    originName é o nome do elemento da cadeia de serviços multimídia onde a conexão será iniciada.

    • type: string
    • required: true

  • 'originType'

    originType é o tipo de elemento (device/vms) da cadeia de serviços multimídia onde a conexão será iniciada.

    • type: string
    • required: true
    • constraints:
      • valid_values: [device, vms]

  • 'PortDestine'

    PortDestine é a porta de destino, esta variável não é necessária para preencher, mas serve como guia para cadeias de serviços multimídia complexas.

    • type: integer
    • required: false

  • 'outputType'

    outputType é o tipo de dado que será enviado ao destino, esta variável é utilizada para verificar se o tipo de fluxo multimídia enviado corresponde ao tipo aceito pelo destino

    • type: string
    • required: true
    • constraints:
      • valid_values: [video, audio, text]

  • 'formatType'

    formatType é o formato dos dados que serão enviados ao destino, esta variável é utilizada para verificar se o formato do fluxo multimídia enviado corresponde ao formato aceito pelo destino.

    • type: string
    • required: false
    • constraints:
      • valid_values: [avi, mp3, mp4 ....]

'destine'

destine agrupa as características de onde a conexão vai terminar.

  • 'destineName'

    destineName é o nome do elemento da cadeia de serviços multimédia para onde será enviado o stream multimédia.

    • type: string
    • required: true

  • 'destineIp'

    destinationIp é o endereço IP para onde o stream multimídia será enviado, esta informação é essencial caso você queira gerenciar manualmente a alocação de endereços IP em um ambiente virtualizado.

    Na arquitetura alfa, esta variável é atribuída pelo orquestrador, portanto não é necessário preenchê-la, a menos que seja o destino final da cadeia de serviços multimídia, neste caso o endereço de destino seria um endereço IP de um computador dentro dá rede.

    • type: lPrism.datatype.vms.AddressData
    • required: false

  • 'destinePort'

    destinePorts é a porta de destino da conexão, esta informação pode vir por padrão dos desenvolvedores da aplicação (na maioria dos casos), ou os desenvolvedores podem deixar para configurar esta variável na inicialização do VMS (startupParameters).

    • type: integer
    • required: true

  • 'inputType'

    inputType é o tipo de dado que o destino recebe.Esta variável é usada para verificar se o tipo de stream multimídia recebido corresponde ao aceito pelo VMS.

    • type: string
    • required: true
    • constraints:
      • valid_values: [video, audio, text]

Exemplo de implementação (YAML)

  virtualLinks:
  -
      origin:
        originName: example1 video ball #Name of origin connect
        originType: device #Type of origin 
        PortDestine: 5000 #optional 
        outputType: video # output media stream type (vide, audi, text) 
        formatType: #optional format tipe of midia stream
      destine:
        destineName: vms1 example 1 # Destination name
        destineIp: # Configure for orchestater 
        destinePort: 5000 #information of VMS
        inputType: video # input media stream type 
  -
      origin:
        originName: vms1 example 1 #Name of origin connect
        originType: vms #Type of origin 
        PortDestine: 20002 #optional 
        outputType: video # output media stream type (vide, audi, text) 
        formatType: #optional format tipe of midia stream
      destine:
        destineName: 192.168.0.103 # Destination name
        destineIp: 192.168.0.103 #IP of the final destination where the multimedia stream will be published
        destinePort: 20002 #Developer Information 
        inputType: video # input media stream type
